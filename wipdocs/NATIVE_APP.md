# Native iOS & Android App (Capacitor)

The Music Player frontend is wrapped with [Capacitor](https://capacitorjs.com) to produce native iOS and Android apps from the same Nuxt/Vue codebase. The primary motivation is **background audio playback** — iOS PWAs pause audio when the app is backgrounded or the device is locked.

## Architecture

```
frontend/
├── capacitor.config.ts          # Capacitor project config
├── app/
│   ├── utils/platform.ts        # isNative(), isIOS(), isAndroid(), isWeb()
│   ├── composables/
│   │   ├── useNativeAudio.ts    # Background audio lifecycle
│   │   └── useMediaSession.ts   # Lock screen metadata & controls
│   └── services/
│       └── http.service.ts      # resolveBaseURL() for native API routing
├── ios/                         # Xcode project (generated by Capacitor)
│   └── App/App/Info.plist       # UIBackgroundModes → audio
└── android/                     # Android Studio project (generated by Capacitor)
    └── app/src/main/AndroidManifest.xml
```

### How it works

Capacitor bundles the static HTML/JS/CSS output from `nuxt generate` into a native WebView. The native shell provides:

- **Background audio** via `UIBackgroundModes: audio` (iOS) and `FOREGROUND_SERVICE_MEDIA_PLAYBACK` (Android)
- **App lifecycle events** via `@capacitor/app` — the `useNativeAudio` composable listens for background/foreground transitions and auto-resumes playback after interruptions (phone calls, Siri, etc.)
- **Lock screen controls** via the existing Media Session API (works in both web and native)

### API routing in native builds

When running as a native app, the WebView loads from local files — not from the backend server. API requests need an absolute URL. The `resolveBaseURL()` function in `http.service.ts` handles this:

1. Checks for a user-configured `backendURL` in `localStorage` (set via a settings page)
2. Falls back to the `NUXT_PUBLIC_BACKEND_URL` env var baked in at build time
3. Falls back to `/` (relative, for web builds)

Set the backend URL at build time:

```bash
NUXT_PUBLIC_BACKEND_URL=https://music.example.com yarn generate
```

Or at runtime by writing to `localStorage`:

```js
localStorage.setItem("backendURL", "https://music.example.com");
```

## Build Commands

### Makefile (from project root)

| Target              | Description                                   |
|---------------------|-----------------------------------------------|
| `make cap-build`    | Generate frontend + sync to iOS and Android   |
| `make cap-sync`     | Sync web assets only (skip frontend rebuild)  |
| `make cap-open-ios` | Open iOS project in Xcode                     |
| `make cap-open-android` | Open Android project in Android Studio    |
| `make cap-run-ios`  | Build + run on iOS (runs `cap-build` first)   |
| `make cap-run-android` | Build + run on Android (runs `cap-build` first) |

### Yarn scripts (from `frontend/`)

| Script              | Description                                    |
|---------------------|------------------------------------------------|
| `yarn cap:build`    | `nuxt generate && npx cap sync`                |
| `yarn cap:sync`     | `npx cap sync`                                 |
| `yarn cap:open:ios` | Open Xcode                                     |
| `yarn cap:open:android` | Open Android Studio                        |
| `yarn cap:run:ios`  | Build and run on iOS device/simulator          |
| `yarn cap:run:android` | Build and run on Android device/emulator    |

## Typical Development Workflow

```bash
# 1. Make code changes in frontend/app/

# 2. Build and sync
make cap-build

# 3. Open in IDE and run on device
make cap-open-ios     # or cap-open-android
# In Xcode: select device → Play (Cmd+R)
# In Android Studio: select device → Run
```

For iterating quickly, you can also point the native app at your dev server by uncommenting the `url` field in `capacitor.config.ts`:

```ts
server: {
  url: "http://YOUR_LOCAL_IP:3000/ui/",
}
```

Then changes are live without rebuilding — just refresh. Remove this before building for production.

## Platform Configuration

### iOS — `ios/App/App/Info.plist`

Key entries:

| Key                    | Value    | Purpose                                       |
|------------------------|----------|-----------------------------------------------|
| `UIBackgroundModes`    | `audio`  | Allows audio playback when app is backgrounded |
| `CFBundleDisplayName`  | `Music Player` | App name shown on home screen            |

### Android — `android/app/src/main/AndroidManifest.xml`

Key permissions:

| Permission                               | Purpose                              |
|------------------------------------------|--------------------------------------|
| `INTERNET`                               | Network access for API/streaming     |
| `WAKE_LOCK`                              | Keep CPU awake during playback       |
| `FOREGROUND_SERVICE`                     | Run background service               |
| `FOREGROUND_SERVICE_MEDIA_PLAYBACK`      | Media playback foreground service (Android 14+) |

### Capacitor Config — `frontend/capacitor.config.ts`

| Field       | Value               | Notes                                    |
|-------------|----------------------|------------------------------------------|
| `appId`     | `com.musicplayer.app` | Bundle identifier for both platforms    |
| `appName`   | `Music Player`       | Display name                             |
| `webDir`    | `.output/public`     | Output of `nuxt generate`                |

## Outstanding TODOs

### Required before first device test

- [ ] **Set backend URL for native builds** — The app needs to know where the backend server lives. Either set `NUXT_PUBLIC_BACKEND_URL` at build time or add a settings page that writes to `localStorage`.
- [ ] **Configure code signing (iOS)** — In Xcode, go to Signing & Capabilities and select your Apple Developer team. Required to run on a real device.
- [ ] **Test background audio on a real device** — Simulator doesn't fully replicate background audio behavior. Verify music continues playing after locking the device and switching apps.

### Required before App Store / Play Store submission

- [ ] **Generate app icons** — Currently using Capacitor defaults. Generate proper icons from the waveform logo:
  ```bash
  cd frontend
  npm install --save-dev @capacitor/assets
  npx capacitor-assets generate --iconBackgroundColor "#1a1a1a" --iconBackgroundColorDark "#1a1a1a"
  ```
- [ ] **Generate splash screens** — Same tool as icons, needs a source image at `assets/splash.png` (2732x2732).
- [ ] **Create Apple Developer account** — $99/year at [developer.apple.com](https://developer.apple.com). Required for App Store distribution and TestFlight.
- [ ] **Create Google Play Developer account** — $25 one-time fee at [play.google.com/console](https://play.google.com/console).
- [ ] **Add a settings page for backend URL** — Native app users need a way to configure which server to connect to. Write to `localStorage.setItem("backendURL", url)`.
- [ ] **Handle WebSocket connections in native mode** — The current dev proxy config bypasses WS through Vite. In native builds, WebSocket URLs need to resolve to the absolute backend URL. Verify `/api/ws/player` and `/api/ws/control/:id` connections work.
- [ ] **Privacy policy / terms of service** — Required by both app stores.
- [ ] **App Store screenshots** — Required for submission. Generate for iPhone 6.7", 6.5", iPad 12.9".

### Nice to have

- [ ] **Push notifications** — Notify when a remote-controlled device starts/stops playback. Would need `@capacitor/push-notifications`.
- [ ] **Offline support** — Cache recently played tracks for offline playback. The service worker (`sw.js`) already caches static assets but not media files.
- [ ] **Deep linking** — Open specific tracks/playlists via URL scheme (`musicplayer://track/123`). Capacitor supports this via `@capacitor/app` URL open events.
- [ ] **Capacitor live reload in dev** — Uncomment `server.url` in `capacitor.config.ts` and point to your dev server IP for hot-reload during native development.
- [ ] **Automated CI builds** — Add GitHub Actions or similar to build iOS (via `xcodebuild`) and Android (via `gradle`) archives on push to main.
